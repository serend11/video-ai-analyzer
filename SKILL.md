# Video AI Analyzer v2

Multi-provider AI-driven video understanding: extract frames, analyze with vision AI, transcribe speech, and produce a comprehensive structured report — all in one pipeline.

## 🚀 Quick Start

```bash
# Set your API key (depending on provider)
export OPENAI_API_KEY="sk-..."      # for openai / openai-compatible
export ANTHROPIC_API_KEY="sk-ant-..." # for anthropic
export GOOGLE_API_KEY="..."          # for google
# No key needed for ollama (local)

# Run analysis
{baseDir}/scripts/analyze.sh /path/to/video.mp4
```

This produces a full analysis report with auto-generated summary at `./video-analysis-{name}-{timestamp}/report.md`.

## ✨ What's New in v2

- **Multi-Provider**: OpenAI, Anthropic (Claude), Google (Gemini), Ollama (local), any OpenAI-compatible
- **Parallel Frame Analysis**: Configurable concurrency (default 5x) — up to 5× faster
- **Auto-Summary**: Complete summary generated by AI after frame analysis — no more placeholder
- **Cost Estimation**: Shows estimated API call count before processing
- **Retry Logic**: Exponential backoff on 429/5xx errors for reliability
- **Better Validation**: Input validation for all numeric parameters

## 📋 Options

| Flag | Default | Description |
|------|---------|-------------|
| `--provider NAME` | `openai` | AI provider (openai\|anthropic\|google\|ollama\|openai-compatible) |
| `--model MODEL` | provider default | Vision model for frame analysis |
| `--summary-model M` | cheaper variant | Model for final summary generation |
| `--interval N` | `10` | Seconds between frame captures |
| `--max-frames N` | `20` | Max frames to analyze (API cost control) |
| `--out DIR` | `./video-analysis-{ts}` | Output directory |
| `--no-transcribe` | `false` | Skip audio transcription |
| `--language LANG` | auto | Whisper language hint (e.g., `zh`, `en`) |
| `--detail LEVEL` | `low` | Image detail: low\|high\|auto (OpenAI) |
| `--max-tokens N` | `500` | Max tokens per frame analysis |
| `--temperature T` | `0.7` | Sampling temperature |
| `--parallel N` | `5` | Max concurrent frame analyses |
| `--base-url URL` | provider default | Override API base URL |
| `--version` | — | Print version and exit |

## 📖 Examples

```bash
# ─── OpenAI (default) ──────────────────────────────────
{baseDir}/scripts/analyze.sh meeting.mp4

# ─── Anthropic Claude ──────────────────────────────────
{baseDir}/scripts/analyze.sh video.mp4 \
  --provider anthropic \
  --model claude-3-5-sonnet-20241022

# ─── Google Gemini (free tier available) ──────────────
{baseDir}/scripts/analyze.sh lecture.mp4 \
  --provider google \
  --language zh \
  --interval 5

# ─── Local Ollama ─────────────────────────────────────
{baseDir}/scripts/analyze.sh demo.mp4 \
  --provider ollama \
  --model llava

# ─── DeepSeek / Groq / any OpenAI-compatible ──────────
{baseDir}/scripts/analyze.sh clip.mp4 \
  --provider openai-compatible \
  --base-url https://api.deepseek.com \
  --model deepseek-chat

# ─── Long video: sparse sampling ──────────────────────
{baseDir}/scripts/analyze.sh movie.mp4 \
  --interval 60 --max-frames 15 --no-transcribe

# ─── Fine-grained analysis with full parallelism ──────
{baseDir}/scripts/analyze.sh product-demo.mp4 \
  --interval 5 --max-frames 30 --parallel 10 \
  --detail high
```

## 🤖 Agent Instructions

When a user asks to analyze a video, follow these steps:

1. **Check prerequisites**: Ensure `ffmpeg` + `ffprobe` are installed
2. **Choose provider**: Match the provider to the LLM you run on (OpenAI agent → `--provider openai`, Claude agent → `--provider anthropic`, etc.)
3. **Set API key**: Export the appropriate environment variable
4. **Run analysis**: `{baseDir}/scripts/analyze.sh <video> --provider <provider> [options]`
5. **Read results**: Open `report.md` and present findings to the user
6. **Handle errors**: If transcription fails, the report still works. If frame analysis fails, check API key/network.

### Provider Selection Guide for Agents

| Your Agent Type | Use `--provider` | Notes |
|----------------|------------------|-------|
| Running on GPT-4/GPT-4o | `openai` | Fastest, best vision |
| Running on Claude | `anthropic` | Great at detailed descriptions |
| Running on Gemini | `google` | Free tier available |
| Local models | `ollama` | Zero API cost, needs `ollama pull llava` |
| Custom endpoint | `openai-compatible` | Any v1/chat/completions endpoint |

## 🎨 Design Philosophy

- **Multi-Provider**: Works with any LLM ecosystem — not locked to OpenAI
- **Lightweight**: Only ffmpeg + python3. Zero pip dependencies. Zero local model downloads.
- **Focused**: Analysis only. "看懂" (understanding) is the single goal.
- **Comprehensive**: Vision + Audio + AI Summary for complete comprehension
- **Cost-aware**: `--max-frames`, `--interval`, provider choice all control cost
- **Agent-friendly**: Self-contained scripts, clear output structure, cache for resume

## 📝 Notes

- Frame analysis results are cached — re-running picks up where it left off
- Transcription always uses OpenAI Whisper (best quality). Skip with `--no-transcribe` if using non-OpenAI provider
- Ollama requires `ollama pull llava` (or preferred vision model) before first use
- `openai-compatible` provider works with DeepSeek, Groq, LM Studio, vLLM, and any `/v1/chat/completions` endpoint
- For videos >30 min, increase `--interval` or reduce `--max-frames`

## 🔧 Requirements

- **ffmpeg** + **ffprobe**: `brew install ffmpeg` or `apt install ffmpeg`
- **python3**: Standard library only — no pip installs needed
- **API key**: One of `OPENAI_API_KEY`, `ANTHROPIC_API_KEY`, or `GOOGLE_API_KEY` (none for Ollama)
